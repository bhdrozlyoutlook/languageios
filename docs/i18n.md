# Lokalizasyon (i18n)

Uygulama **Türkçe temelli**; İngilizce lokalizasyonun **altyapısı kuruldu ve doğrulandı**
(welcome ekranı İngilizce cihazda tamamen İngilizce görünüyor). Kalan iş **mekanik**:
aşağıdaki desenle metinleri katalogla.

## Kurulum (hazır)

- `project.yml` → `options.developmentLanguage: tr` (temel dil = Türkçe; literal stringler temeldir).
- `App/en.lproj/Localizable.strings` → anahtar = Türkçe literal, değer = İngilizce çeviri.
  Uygulama paketine girer; cihaz dili İngilizce ise bu dosyadan çeviri kullanılır, değilse
  Türkçe literale düşer.

## Desen — bir metni lokalize etmek

1. **Literal `Text("...")`** → otomatik lokalize olur. Sadece `App/en.lproj/Localizable.strings`'e
   `"Türkçe metin" = "English text";` satırı ekle. Kod değişmez.

2. **`String` parametresiyle dolaşan metin** (örn. `Text(title)` — `title: String`) → otomatik
   lokalize **olmaz**. Parametrenin tipini `String` → `LocalizedStringKey` yap; çağrı yerleri
   string literal geçtiği sürece değişmez. Sonra strings dosyasına çeviriyi ekle.
   - Örnek (yapıldı): `OnboardingPrimaryButton.title`, `LessonActionButton.title`.

3. **Model hesaplanmış stringleri** (örn. `TargetLanguage.title` → düz `String` döner, sonra
   `Text(...)`'e girer) → `String(localized:)` kullan veya `LocalizedStringKey` döndür. Bunlar
   en hacimli grup; sırayla çevrilebilir.

## Doğrulama

```bash
xcrun simctl launch <udid> com.bhdrozly.languageios -AppleLanguages "(en)"
```
ile uygulamayı İngilizce başlat ve ekranları kontrol et.

## Durum

- ✅ Altyapı + ilk dilim (welcome, onboarding birincil buton, ders aksiyon butonları, birkaç literal).
- ⏳ Kalan: onboarding adım başlıkları/seçenek etiketleri, model enum stringleri, profil/harita
  metinlerinin geri kalanı — yukarıdaki desenle eklenecek.
